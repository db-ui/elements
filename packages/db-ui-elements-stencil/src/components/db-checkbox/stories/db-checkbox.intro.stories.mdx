<db-headline variant="1">&lt;db-checkbox&gt;</db-headline>

The `<db-checkbox></db-checkbox>` represents a two-state control that represents the element's checkedness state. If the element's checkedness state is true, the control represents a positive selection, and if it is false, a negative selection.

Most of attributes are used in the same way as with a native checkbox. You can try out on <a href="/story/db-checkbox--live-demo">Checkbox Live Demo</a> Page.

<Canvas>
<db-checkbox label="Checkbox" Name="checkbox-name"></db-checkbox>
</Canvas>

<db-headline variant="2">disabled</db-headline>

Make checkbox look inactive by adding the disabled boolean attribute to any &lt;db-checkbox&gt; element.

<Canvas>
<db-checkbox label="Checkbox" Name="checkbox-name" disabled></db-checkbox>
</Canvas>

<db-headline variant="2">indeterminate</db-headline>

Make checkbox look like in an indeterminate state by adding the 'indeterminate' boolean attribute to any &lt;db-checkbox&gt; element.

<Canvas>
<db-checkbox label="Checkbox" Name="checkbox-name" indeterminate="true"></db-checkbox>
</Canvas>

Alternatively, one can directly call `<db-checkbox>.setIndeterminate(boolean)` on the web component to set the state indeterminate state of the checkbox via JavaScript.

<db-headline variant="2">visually hide the label</db-headline>

> A label for a form control helps everyone better understand its purpose. In some cases, the purpose may be clear enough from the context when the content is rendered visually. The label can be hidden visually, though it still needs to be provided within the code to support other forms of presentation and interaction, such as for screen reader and speech input users.

source: <https://www.w3.org/WAI/tutorials/forms/labels/#hiding-label-text>

You could set the attribute `label-hidden="true"` for the rare case that you would want to visually hide the textual label for a checkbox element.

<Canvas>
<db-checkbox label="Checkbox" name="checkbox-name" label-hidden="true"></db-checkbox>
</Canvas>
