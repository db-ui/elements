<Meta title="02-Navigation/Pagination/Intro" />

<db-headline variant="1"> &lt;db-pagination&gt;</db-headline>

The `<db-pagination></db-pagination>` element represents a pagination navigation
element which lets the user select a certain page number.

There are two kinds how one can consume this component.

If anchor elements are passed into the `<db-pagination>` they will be used as
is. The first element is assumed to be the beginning of the collection of pages
in the pagination.
The same is true for the last element but it is assumed that it is your very
last page.
If ellipses are needed one can activate them via the attributes
`ellipsisstart="true"` and `ellipsisend="true"`.

Caveat: Previous and next arrows will always jump from the second/penultimate
element to the first/last element passed as children.

The second kind how this component can be consumed is without children but with
a custom event. When a certain page is selected, `<db-pagination>` emits a custom `dbChange`
event whose `event .detail.target.innerText` value can be read from your
framework of choice or via plain Javascript.
