<Meta title="05-Data-Input/Input/Intro" />

<db-headline variant="1">&lt;db-input&gt;</db-headline>

The input element represents a one line plain text edit control for the element's value. Most of attributes are used in the same way as with a native input. You can try out on <a href="/story/db-input--live-demo">Input Live Demo</a> Page.

<Canvas>
  <db-input label="Semitransparent" name="input-name"></db-input>
  <db-input label="White" name="input-name" variant="white"></db-input>
  <db-input label="Solid" name="input-name" variant="solid"></db-input>
  <db-input label="Outline" name="input-name" variant="outline"></db-input>
</Canvas>

<db-headline variant="2">disabled</db-headline>

Make inputs look inactive by adding the disabled boolean attribute to any &lt;db-input&gt; element.

<Canvas>
  <db-input label="Inputfield" name="input-name" disabled></db-input>
</Canvas>

<db-headline variant="2">description/ hint</db-headline>

Add a description below the &lt;db-input&gt; element.

<Canvas>
  <db-input
    label="Inputfield"
    name="input-name"
    description="A clue, a clue"
  ></db-input>
</Canvas>

<db-headline variant="2">validation</db-headline>

Shows validation on the &lt;db-input&gt; element, needs required attribute.

<Canvas>
  <db-input
    label="Inputfield"
    name="input-name"
    value="Valid"
    required
    minlength={3}
  ></db-input>
  <db-input
    label="Inputfield"
    name="input-name"
    required
    minlength={3}
  ></db-input>
</Canvas>

<db-headline variant="2">list</db-headline>

The list attribute connects the input with a set of options that represent predefined options in a datalist element used for the input field.
