<Meta title="05-Data-Input/Input/Intro" />

<db-headline variant="1">&lt;db-input&gt;</db-headline>

The input element represents a one line plain text edit control for the element's value. Most of attributes are used in the same way as with a native input. You can try out on <a href="/story/db-input--live-demo">Input Live Demo</a> Page.

<Canvas>
  <db-input label="Semitransparent"></db-input>
  <db-input label="White" variant="white"></db-input>
  <db-input label="Solid" variant="solid"></db-input>
  <db-input label="Outline" variant="outline"></db-input>
</Canvas>

<db-headline variant="2">disabled</db-headline>

Make inputs look inactive by adding the disabled boolean attribute to any &lt;db-input&gt; element.

<Canvas>
  <db-input label="Inputfield" disabled></db-input>
</Canvas>

<db-headline variant="2">description/ hint</db-headline>

Add a description below the &lt;db-input&gt; element.

<Canvas>
  <db-input label="Inputfield" description="A clue, a clue"></db-input>
</Canvas>

<db-headline variant="2">validation</db-headline>

Shows validation on the &lt;db-input&gt; element, needs required attribute.

<Canvas>
  <db-input label="Inputfield" value="Valid" required minlength={3}></db-input>
  <db-input label="Inputfield" required minlength={3}></db-input>
</Canvas>

<db-headline variant="2">list</db-headline>

The list attribute connects the input with a set of options that represent predefined options in a datalist element used for the input field.

<Canvas>
  <db-input label="Inputfield" list="datalist">
    <option value="Value 1">Option 1</option>
    <option value="Value 2">Option 2</option>
    <option value="Value 3">Option 3</option>
  </db-input>
</Canvas>

<db-headline variant="2">visually hide the label</db-headline>

> A label for a form control helps everyone better understand its purpose. In some cases, the purpose may be clear enough from the context when the content is rendered visually. The label can be hidden visually, though it still needs to be provided within the code to support other forms of presentation and interaction, such as for screen reader and speech input users.
>
> <a href="https://www.w3.org/WAI/tutorials/forms/labels/#hiding-label-text">
>   source
> </a>

You could set the attribute `label-hidden="true"` for the rare case that you would want to visually hide the textual label for a input element.

<Canvas>
  <db-input label="Semitransparent" label-hidden="true"></db-input>
</Canvas>

<db-headline variant="2">maxlength with counter</db-headline>

On providing the maxlength attribute, a counter will be shown below the input field.

<Canvas>
  <db-input label="maxlength example" maxlength="100"></db-input>
</Canvas>
