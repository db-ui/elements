<Meta title="03-Action/Button/Intro" />

<db-headline variant="1">&lt;db-button&gt;</db-headline>

The `<db-button></db-button>` element represents a clickable button, used to submit forms or anywhere in a document for accessible, standard button functionality. Most of attributes are used in the same way as with a native button. You can try out on <a href="/story/db-button--live-demo">Button Live Demo</a> Page.

<Canvas>
  <db-button>Button</db-button>
</Canvas>

<db-headline variant="2">variant</db-headline>

The variant attribute specifies a visual expression of button.

<Canvas>
  <db-button variant="primary">Primary</db-button><db-button variant="primary">Primary</db-button><db-button variant="secondary">Secondary</db-button><db-button variant="secondary-inverted">Secondary inverted</db-button><db-button variant="ghost">Ghost</db-button>
</Canvas>

<db-headline variant="2">sizes</db-headline>

There are three different sizes of buttons to choose from: small, regular (default), large

<Canvas>
  <db-button variant="primary" size="small">Primary small</db-button><db-button variant="primary">Primary</db-button><db-button variant="primary" size="large">Primary large</db-button>
</Canvas>

<db-headline variant="2">disabled</db-headline>

Make buttons look inactive by adding the disabled boolean attribute to any `<db-button>` element.

<Canvas>
  <db-button variant="primary" disabled>Primary</db-button><db-button variant="tertiary" disabled>Tertiary</db-button><db-button variant="secondary" disabled>Secondary</db-button>
</Canvas>

<db-headline variant="2">icons</db-headline>

You could add an icon to the front of the text by using the `icon` attribute and defining an icon by it's identifier (like e.g. download, compare to [DB UI Icons](https://db-ui.github.io/core/patterns/base-icons/index.html)).

<Canvas>
  <db-button variant="primary" icon="download">Primary</db-button>
  <db-button variant="tertiary" icon="download">Tertiary</db-button>
  <db-button variant="secondary" icon="download">Secondary</db-button>
</Canvas>

<db-headline variant="2">icononly (only use with care)</db-headline>

If you'd only like to display the icon without any text, you could set the attribute `icononly` on the button component. But please keep in mind that a visual text ensures clarity on the subject of this button as well enables use cases like controlling the page by voice (accessibility).

<Canvas>
  <db-button variant="primary" icon="download" icononly>Button</db-button><db-button variant="tertiary" icon="download" icononly>Button</db-button><db-button variant="secondary" icon="download" icononly>Button</db-button>
</Canvas>

<db-headline variant="2">loading state</db-headline>

If you'd only like to define that the buttons context (like e.g. a form) is in a loading state, you could set the attribute `state="loading"` on the button component.

<Canvas>
  <db-button variant="primary" state="loading">Button</db-button><db-button variant="tertiary" state="loading">Button</db-button><db-button variant="secondary" state="loading">Button</db-button>
</Canvas>
